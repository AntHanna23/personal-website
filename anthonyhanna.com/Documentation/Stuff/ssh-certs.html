Creating Certificate Authority

For starters we need to identify which server we would like to be our certificate authority(CA) or more where the CA key are going to be stored. Once that decision is made create keys using ssh-keygen like you normally would when creating SSH keys. Because of the nature of this key give it a strong password.

Configuring Remote Server

To any server that you would like users to be able to connect to, the CA public key must be placed in the /etc/ssh/ file and the sshd_config must be edited. Add the line TrustedUserKeys /etc/ssh/ca_key.pub to the sshd_config file. Now restart ssh and your server is now configured.

Signing the Users SSH Key

Lastly we must sign the keys before we distribute them so that the sever knows they are valid. Create a key for a new user. Once this is created run ssh-keygen -s ca_key -I users_name -n account_names,root -V -5m:+3650d user_key.pub . Now send the generated certificate along with the keys to the user and have them add these keys to memory. The user now can access any server that has the CA public key on it
